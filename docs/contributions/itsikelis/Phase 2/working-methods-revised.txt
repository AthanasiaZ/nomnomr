 Η ομάδα θα έχει έναν Υπεύθυνο Έργου (διαφορετικό σε κάθε παραδοτέο, έτσι ώστε να αποκτήσουν όλα τα μέλη εμπειρία από team management), δουλειά του οποίου είναι να καθορίζει τα tasks για το εκάστοτε παραδοτέο, να δημιουργεί τις καρτέλες στον πίνακα της Kanban και να ισομοιράζει τα tasks στα μέλη, όπως κρίνει αυτός, σύμφωνα με τις ικανότητες και τον απαιτούμενο χρόνο διεκπεραίωσης του task. Η προτεραιότητα που θα βάλουν τα μέλη στις εργασίες που τους έχουν ανατεθεί, αφήνεται στην δική τους επιμέλεια, προς αποφυγή πρόσθετης πολυπλοκότητας και μεγέθυνσης του learning curve για τα μέλη που δεν έχουν ασχοληθεί στο παρελθόν με Agile μεθόδους.

 Για μείωση των bottlenecks στο column Testing/Review της Kanban, μιας και μπορεί να υπάρχει δυσκολία ομαδικής συνεύρεσης και ανάλυσης της δουλειάς από κάθε μέλος ξεχωριστά, αποφασίστηκε να δημιουργηθεί ένας ρόλος Quality Assurance Manager. Σε αυτόν, ανατίθεται η δουλειά του ελέγχου ποιότητας της δουλειάς του κάθε μέλους, με βάση τις παρατηρήσεις του ιδίου και των διδασκόντων. Αυτός θα ενημερώνει τα μέλη σχετικά με την ποιότητα της δουλειάς τους και θα προτείνει αλλαγές.

 Για την συγγραφή του κώδικα και την ανάρτηση της δουλειάς στο GitHub, αποφασίστηκε να χρησιμοποιηθεί η μέθοδος GitFlow. Η επιλογή αυτή έγινε λόγω της απλότητας εκμάθησης της μεθόδου αυτής από μέλη που δεν έχουν ασχοληθεί με Source Control στο παρελθόν, καθώς και για να αποφεύγονται περίοδοι που το main branch θα περιέχει μη-λειτουργικό, μη ελεγμένο κώδικα. Μόλις ανατίθεται ενα νέο task συγγραφής κώδικα σε κάποιο μέλος, αυτό θα δημιουργεί ένα branch στο οποίο θα αναπτύσσει τον κώδικα που του ζητήθηκε, θα διενεργεί τους απαραίτητους ελέγχους και, μόλις το task ολοκληρωθεί, θα κάνει merge με το main branch.
